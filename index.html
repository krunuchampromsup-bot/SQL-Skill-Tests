SQL VIEW Master Class

1. รูปแบบคำสั่งการสร้าง VIEW

VIEW คือตารางเสมือนที่สร้างจากคำสั่ง SELECT เพื่อช่วยให้การจัดการข้อมูลง่ายขึ้น

CREATE VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

ตัวอย่างการใช้งาน: สร้าง View เพื่อดูเฉพาะพนักงานแผนก IT

CREATE VIEW it_staff AS SELECT name, salary FROM employees WHERE dept = 'IT';

ผลลัพธ์ที่ได้:

namesalary
Somsak45,000
Wichai38,000

2. การเพิ่มข้อมูลผ่าน VIEW

เราสามารถเพิ่มข้อมูลลงในตารางหลักผ่านทาง VIEW ได้หาก VIEW นั้นประกอบด้วยคอลัมน์ที่จำเป็นครบถ้วน

INSERT INTO view_name (column1, column2)
VALUES (value1, value2);

ตัวอย่างการใช้งาน: เพิ่มพนักงานใหม่ผ่าน it_staff

INSERT INTO it_staff (name, salary) VALUES ('Jane', 42000);

ผลลัพธ์ในตาราง (รวมข้อมูลใหม่):

namesalary
......
Jane42,000

3. การแก้ไขข้อมูลผ่าน VIEW

การแก้ไขข้อมูลใน VIEW จะส่งผลกระทบโดยตรงไปยังข้อมูลที่อยู่ในตารางหลัก

UPDATE view_name
SET column1 = value1
WHERE condition;

ตัวอย่างการใช้งาน: ปรับเงินเดือนพนักงานชื่อ Somsak ผ่าน View

UPDATE it_staff SET salary = 50000 WHERE name = 'Somsak';

ผลลัพธ์หลังการแก้ไข:

namesalary
Somsak50,000
Wichai38,000

4. การลบข้อมูลผ่าน VIEW

ใช้ลบข้อมูลในตารางหลักที่ตรงตามเงื่อนไขที่ VIEW มองเห็น

DELETE FROM view_name
WHERE condition;

ตัวอย่างการใช้งาน: ลบข้อมูลพนักงานชื่อ Wichai ออกจากระบบ

DELETE FROM it_staff WHERE name = 'Wichai';

ผลลัพธ์หลังการลบ:

namesalary
Somsak45,000
(ข้อมูล Wichai ถูกลบแล้ว)
